# 1. Objective

The objective of this test plan is to verify that all functional and non-functional requirements of the CloudVault application are working as expected before release.



# 2. Scope

### In Scope

* User Registration
* User Login
* Dashboard
* File Upload
* File Download
* Rename File
* Delete File
* Search File
* Folder Management
* User Profile
* Logout

### Out of Scope

* Mobile application testing
* Performance testing
* Security penetration testing
* Third-party integrations



# 3. Test Strategy

The following testing types will be performed:

* Functional Testing
* Smoke Testing
* Sanity Testing
* Regression Testing
* UI Testing
* Negative Testing
* API Testing
* Database Validation



# 4. Test Environment

| Component       | Details         |
| --------------- | --------------- |

| Database        | MySQL           |
| API Tool        | Postman         |
| Documentation   | Microsoft Excel |
| Version Control | Git & GitHub    |



# 5. Entry Criteria

* BRD approved.
* SRS approved.
* Test environment is available.
* Application build is deployed.



# 6. Exit Criteria

* All planned test cases executed.
* Critical and high-severity defects resolved.
* Test summary report prepared.
* Client approval received.



# 7. Deliverables

* BRD
* SRS
* Test Plan
* Test Scenarios
* Test Cases
* Bug Report
* RTM
* Test Execution Report
* API Test Report
* SQL Validation Report
* Test Summary Report



# 8. Risks

* Requirement changes during testing.
* Environment instability.
* Delay in bug fixes.
* Limited testing time.
