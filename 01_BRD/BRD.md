# 1. Introduction

CloudVault is a secure cloud storage web application that enables users to upload, download, organize, search, rename, and delete files securely. The application provides authenticated users with a reliable platform to store and manage their files online.



# 2. Business Objectives

* Provide secure user authentication.
* Allow users to upload and manage files online.
* Enable efficient file organization.
* Protect user data from unauthorized access.
* Deliver a simple and user-friendly experience.



# 3. Project Scope

The application supports the following features:

* User Registration
* User Login
* Dashboard
* File Upload
* File Download
* Rename File
* Delete File
* Search File
* Folder Management
* User Profile
* Logout



# 4. Stakeholders

* Client
* Business Analyst
* Project Manager
* Development Team
* QA Team
* End Users



# 5. User Roles

## Registered User

A registered user can:

* Register
* Login
* Upload files
* Download files
* Rename files
* Delete files
* Search files
* Create folders
* Update profile
* Logout



# 6. Business Requirements

| Requirement ID | Requirement                                            |
| -------------- | ------------------------------------------------------ |
| BR-001         | User should be able to register using valid details.   |
| BR-002         | User should be able to log in using valid credentials. |
| BR-003         | User should be able to upload files.                   |
| BR-004         | User should be able to download files.                 |
| BR-005         | User should be able to rename uploaded files.          |
| BR-006         | User should be able to delete uploaded files.          |
| BR-007         | User should be able to search uploaded files.          |
| BR-008         | User should be able to update profile information.     |
| BR-009         | User should be able to log out securely.               |



# 7. Assumptions

* Users have an active internet connection.
* Users access the application through a supported web browser.
* Each user has a unique email address.



# 8. Constraints

* Maximum upload size: 100 MB.
* Supported file types: PDF, DOCX, XLSX, JPG, PNG, MP4, ZIP.
* Only authenticated users can access file management features.



# 9. Out of Scope

The following features are not included in Version 1.0:

* File sharing with other users.
* Third-party cloud storage integration.
* Video conferencing.
* Email integration.
* Mobile application support.
